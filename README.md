Eth communication project for stm32f746g-disco :

1\ - Global architecture : 

The project contains the followings folders :

 - ETH : This folder contains the necessary functions to initialize the ethernet configuration. It contains initialization of MAC, DMA,
   RX & TX descriptors plus the IP/UDP header. It also provides functions to manipulate the payload received or to be transmitted.
   
 - GPIO : This folder contains the GPIO initialization to configure the pins in RMII mode (Reduced media Independent Interface).
   
 - Interrupt : This folder contains the parameters to enable interrupts for incoming frames.

 - RCC & SYSCFG : These folders contains the material configuration used by the boot the enable the required peripherals & the RMII mode

 - Timer : This folder contains the functions used to manipulate the timer

 - Startup : Contains the assembly file auto-generated by ST for stm32f746g-disco

 - boot : First part on the main program. It uses parts of previous functions to set the Hadware and Ethernet configuration.

 - main : Contains the main loop of the app, it checks if a command has been received. If yes, the app starts sending frames until the number of frames in input
 has been reached unless it's stopped before.

2\ - How to use : 

Once the binary is flashed, the chip enters an infinite functional loop, and wait for an Ethernet command.
The ethernet command must target the UDP dest Port 0x1103, and have the followings parameters :

 - Number of frames to send ( unsigned 32 bits )
 - Interval between frames in microseconds ( unsigned 32 bits )
 - Stop (0/1) ( ignored if nothing is running ) ( unsigned 32 bits )

Once the command is received it starts to send 50 bytes ethernet frames with a 'PING' and a message counter. It can be stopped by sending
a command again with stop = 1. Other values mean continue.

3\ Technical limits and choices.

In the Ethernet module, the choice was made to have only one TX and one RX descriptor. This is sufficient for the intended use case. Frames are manually sent 
through Colasoft over Ethernet, so the possibility to loose frames is null. For a general use, multiple descriptors must be used.

Timer is set with the default frequency, (16 MHz). Final frequency timer is calculate by Timer_freq =  (Freq_clock / (prescaler + 1 ))
Calculations are made to set the input delay for a hundred timer ticks, and 16 MHz was enough to handle this timing. A minimum interval of 100 µs was set because of 
instability below this timing. Only timer 2 to 5 are declared in the Timer API because the others were not needed. Timer 2 was selected, because it has a 32 bit counter
instead of 16 for 3 & 4. Timer 5 could also have been selected, because it's also a 32 bits counter.

RMII over MII interface has been chosen for the gpio ethernet configuration due the reduced number of required pins : 7 pins for RMII vs 25 for MII.


4\ - Potential improvements :

The low power mode could be implemented to allow CPU sleeping. The MAC filters have to be configured to see frames on the desired UDP dst PORT. We also 
need to set the corresponding interruption in the ETH_Wakeup handler. Then when work is over the chip could go back in sleep mode. 

The LCD screen could also be activated to inform user of the current state, received commands, and number of frames sent.
